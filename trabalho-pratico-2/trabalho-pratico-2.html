📘 Descrição Geral

Neste trabalho prático, você irá:

  1. Provisionar uma VM na AWS Academy
  2. Instalar o Mininet e o FRRouting (FRR)
  3. Executar um script Python que cria dinamicamente uma topologia contendo:
    • 2 roteadores FRR
    • 2 switches L2
    • 2 hosts
  4. Descobrir e configurar manualmente os comandos BGP necessários para:
    • estabelecer a sessão BGP entre r1 e r2
    • anunciar as redes conectadas
    • permitir que h1 pingue h2

Importante:
A topologia não existe previamente.
Todos os roteadores, switches e hosts são criados somente quando o aluno executa o script Python (bgp-topo.py).
Nada é configurado fora do script.

🟦 0. Provisionar a VM na AWS Academy

Crie uma instância EC2 que servirá como ambiente de laboratório.

Recurso Valor

Disco 20 GB
SO Ubuntu Server 24.04 LTS

🟩 Requisitos 

sudo apt update
sudo apt install -y git curl nano python3-pip

🟦 1) Instalar o Mininet 

sudo apt install -y mininet
mn --version

🟧 2) Instalar o FRRouting (FRR) 

curl -s https://deb.frrouting.org/frr/keys.asc | sudo gpg --dearmour -o /usr/share/keyrings/frrouting.gpg

echo "deb [signed-by=/usr/share/keyrings/frrouting.gpg] https://deb.frrouting.org/frr noble frr-stable" \
| sudo tee /etc/apt/sources.list.d/frr.list

sudo apt update
sudo apt install -y frr frr-pythontools

vtysh -c "show version"

🟦 3) Criar a topologia Mininet

O professor fornecerá o arquivo bgp-topo.py, contendo:

  • criação da topologia
  • roteadores FRR com namespaces isolados
  • switches L2
  • hosts configurados
  • blocos BGP incompletos, que os alunos devem preencher

Reforço:
A topologia só é criada quando o aluno executa o script Python.
Antes disso, nenhum roteador, switch ou host existe no sistema.

A topologia utilizada será:

h1 --- s1 --- r1 --- r2 --- s2 --- h2

Endereçamento IPv4

Dispositivo Interface Endereço IP
r1 r1-eth1 10.0.1.1/24
r1 r1-eth2 10.0.12.1/24
r2 r2-eth1 10.0.2.1/24
r2 r2-eth2 10.0.12.2/24
h1 h1-eth0 10.0.1.100/24
h2 h2-eth0 10.0.2.100/24

Sistemas Autônomos (ASNs)

A rede está organizada da seguinte forma:

  • AS 65001: Contém o roteador r1, switch s1 e host h1 (Rede 10.0.1.0/24).

  • AS 65002: Contém o roteador r2, switch s2 e host h2 (Rede 10.0.2.0/24).

Link de Peering: Conexão direta entre r1 e r2 na rede 10.0.12.0/24.

🟨 4) Executar o script 

chmod +x bgp-topo.py
sudo ./bgp-topo.py

Isso abrirá o CLI do Mininet.

🟦 5) ATIVIDADE: Completar as configurações BGP

Você deverá descobrir quais comandos BGP precisam ser adicionados aos arquivos (variáveis):

  • r1_conf
  • r2_conf

para que:

  1. A sessão BGP entre os roteadores r1 e r2 seja estabelecida
  2. As redes conectadas aos hosts sejam anunciadas
  3. O host h1 consiga pingar o host h2 através do roteamento BGP

Os blocos BGP dentro do script estão assim:

r1_conf 

enable
configure terminal
# COMPLETE AQUI

r2_conf 

enable
configure terminal
# COMPLETE AQUI

🎯 Seu objetivo é descobrir quais comandos devem ser adicionados para:

  • iniciar o processo BGP
  • configurar router-id
  • configurar vizinhos
  • ativar o address-family IPv4
  • anunciar as redes locais
  • permitir troca de rotas entre AS diferentes
  • ativar o vizinho dentro do address-family

🧪 6) Verificações

Para que a atividade seja considerada concluída, você deve garantir:

✔️ A sessão BGP entre r1 e r2 deve ser estabelecida

Verifique com:

r1 vtysh -c "show bgp summary"

✔️ r1 deve aprender a rota 10.0.2.0/24

✔️ r2 deve aprender a rota 10.0.1.0/24

Verifique com:

r2 vtysh -c "show bgp"

✔️ h1 deve conseguir pingar h2 

mininet> h1 ping 10.0.2.100

💡 7) Pistas (opcionais)

Use estas pistas apenas se necessário:

  • O vizinho BGP de r1 é 10.0.12.2
  • O vizinho BGP de r2 é 10.0.12.1
  • As redes a serem anunciadas são:
    • r1 → 10.0.1.0/24
    • r2 → 10.0.2.0/24
  • O FRR moderno exige uma configuração extra para permitir troca de rotas entre AS diferentes
  • A ativação do vizinho ocorre dentro do address-family
